## Тема: Книжный интернет-магазин
Индивидуальный проект — паттерн проектирования Strategy (Стратегия).

Автор: Устинов Егор, ПИ-251

## Постановка задачи
Необходимо разработать книжный интернет-магазин, который позволяет:

1.просматривать каталог книг;

2.добавлять книги в корзину;

3.оформлять заказ с разными способами оплаты;

4.добавлять новые способы оплаты без изменения основного кода.

Реализация: консольное приложение на C#.

Реальный сценарий применения
Небольшой книжный магазин продает книги онлайн. Покупатели хотят оплачивать заказ разными способами: кто-то использует кредитную карту, кто-то пользуется ЮMoney.

## Решение:
1.Один раз создается каталог книг.

2.Покупатель выбирает книги и добавляет их в корзину.

3.При оформлении заказа покупатель выбирает способ оплаты.

4.Система автоматически применяет выбранный способ оплаты.

5.При появлении нового способа оплаты его можно добавить без изменения остального кода.

Так магазин быстро адаптируется под потребности покупателей.

## Паттерн Strategy (Стратегия)
Зачем нужен
Способы оплаты могут меняться. Если писать код для каждого способа отдельно внутри класса заказа, то при добавлении нового способа придется менять этот класс. Это рискованно и долго.
Strategy позволяет вынести каждый способ оплаты в отдельный класс и взаимозаменяемо использовать их.

Какую проблему решает в проекте
Проблема	Решение через Strategy
Код заказа знает про все способы оплаты	Заказ работает с интерфейсом, а не с конкретными классами
Добавление нового способа ломает старый код	Новый способ просто реализует интерфейс
Трудно тестировать каждый способ отдельно	Каждый способ оплаты тестируется независимо
Ключевые элементы
IPaymentMethod — интерфейс с методом Pay().

CreditCardPayment — конкретная стратегия оплаты кредитной картой.
YooMoneyPayment — конкретная стратегия оплаты ЮMoney.
Order — контекст, который использует стратегию оплаты.
Program — клиентский код, который выбирает нужную стратегию.

text

Пункт 1 — просмотр каталога
Показывает список доступных книг с ценами.

Пользователь вводит номер книги из каталога. Книга добавляется в корзину.

Завершение работы программы.
